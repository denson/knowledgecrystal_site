# Knowledge Crystal - Jekyll Site Setup

A personal portfolio site for Denson, an AI Engineer specializing in reliable LLM and GraphRAG systems.

**Live at:** https://knowledgecrystal.com

## Quick Start

### Local Development

1. **Install Ruby and Jekyll:**
   ```bash
   # macOS with Homebrew
   brew install ruby

   # Or use rbenv (recommended)
   rbenv install 3.2.0
   rbenv local 3.2.0
   ```

2. **Install dependencies:**
   ```bash
   bundle install
   ```

3. **Start local server:**
   ```bash
   bundle exec jekyll serve
   ```

   Visit `http://localhost:4000` in your browser.

4. **Build for production:**
   ```bash
   bundle exec jekyll build
   ```

   Output is in `_site/` directory.

## Design System

### Colors
- **Background**: `#1a1a2e` (dark, warm)
- **Primary Accent**: `#f59e0b` (amber/orange)
- **Accent Hover**: `#fbbf24` (lighter amber)
- **Text**: `#e8e8f0` (light, warm)
- **Muted Text**: `#999` or `#c0c0c8`

### Typography
- **Font**: System fonts (-apple-system, BlinkMacSystemFont, Segoe UI, Roboto, etc.)
- **Headings**: Bold, 700 weight, warm light color
- **Body**: 16px, 1.6 line-height

### Components
- **Cards**: `#242438` background, `border-radius: 12px`, hover lift effect
- **Buttons**: Amber primary, outlined secondary
- **Navigation**: Sticky top, warm accent on hover/active

### Responsive
- Mobile-first design
- Grid layouts adjust for screens < 768px
- Navigation maintains full functionality on mobile

## Adding Content

### New Blog Post

1. Create file in `_posts/` with format: `YYYY-MM-DD-slug.md`
2. Add frontmatter:
   ```yaml
   ---
   layout: post
   title: "Your Post Title"
   date: 2026-04-05
   excerpt: "Short summary for preview"
   ---
   ```
3. Write content in Markdown

### New Project

1. Create file in `_projects/` with Markdown frontmatter
2. Use template:
   ```yaml
   ---
   layout: project
   title: "Project Title"
   description: "Short description"
   tags: [tag1, tag2]
   link: "https://link"
   ---
   ```

### Update Main Pages

- `index.md` - Home page hero and featured sections
- `about.md` - Bio, skills, background
- `projects.md` - Project grid
- `blog.md` - Blog index

## Deployment to GitHub Pages

1. **Create GitHub repository** named `knowledgecrystal.com` or similar
2. **Enable GitHub Pages** in repository settings:
   - Source: `main` branch, `/root` folder
3. **Update `_config.yml`** if needed:
   ```yaml
   url: "https://knowledgecrystal.com"
   baseurl: ""
   ```
4. **Add DNS records** for custom domain:
   - A records pointing to GitHub Pages IPs
   - CNAME record in `CNAME` file (already set)
5. **Push to main branch**:
   ```bash
   git add .
   git commit -m "Initial site"
   git push origin main
   ```

GitHub will automatically build and deploy the site.

## Performance Notes

### CSS
- All CSS is **inline in `default.html`** for fast page loads

## Troubleshooting

### Site won't build locally
```bash
rm -rf _site .jekyll-cache Gemfile.lock
bundle install
bundle exec jekyll serve
```

### GitHub Pages not updating
- Force refresh: Ctrl+Shift+R or Cmd+Shift+R
- Check GitHub Actions in repo for build errors
- Verify `_config.yml` and file names are correct

### Links broken
- Use relative paths: `{{ '/about.html' | prepend: site.baseurl }}`
- Don't hardcode domain names in links
- Always use Jekyll liquid tags for internal links
